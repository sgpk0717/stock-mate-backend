## 1. 알파 팩터란?

**알파 팩터**란, 주가의 미래 수익률을 예측하는 데 도움이 되는 수학적 수식입니다.

예시:
- `rsi` — RSI(상대강도지수) 값 자체를 팩터로 사용
- `volume_ratio - sma_20 / close` — 거래량 비율과 이동평균 기반 복합 수식
- `log(Abs(rsi - 50)) * volume_ratio` — 비선형 변환을 적용한 고급 수식

**좋은 알파 팩터**의 조건:
- **IC (정보계수)가 높음** — 팩터 값과 다음 날 수익률의 상관관계가 높음 (0.03 이상이면 의미 있음)
- **ICIR이 높음** — IC가 일관되게 높음 (IC의 변동이 적음)
- **인과적으로 유효** — 단순 통계적 우연이 아니라 실제 인과 관계가 있음

Alpha Lab은 **AI(Claude)**가 자동으로 수천 개의 팩터 후보를 생성하고, 통계적으로 검증하고, 유전 진화시켜서 좋은 팩터를 찾아주는 시스템입니다.

## 2. Alpha Lab 진입하기

### 화면 접근 경로

1. 브라우저에서 Stock Mate 앱에 접속합니다 (`http://localhost:5173`)
2. **왼쪽 사이드바**에서 **"알파 탐색"** 메뉴를 클릭합니다
3. Alpha Lab 페이지가 열리며, 상단에 **3개 탭**이 표시됩니다

### 3개 탭 구조

| 탭 | 이름 | 용도 | 난이도 |
|----|------|------|--------|
| 1 | **탐색** | 수동으로 알파 마이닝 실행 + 결과 확인 | 초급 |
| 2 | **공장** | 24시간 자동 팩터 탐색 + 유전 진화 | 중급 |
| 3 | **포트폴리오** | 여러 팩터를 합성하여 복합 팩터 생성 | 고급 |

처음 사용하는 분은 **"탐색" 탭**부터 시작하세요.

## 3. 탐색 탭 — 수동 알파 마이닝

### 화면 구성

탐색 탭은 **좌측 패널(설정)** + **우측 패널(결과)** 2열 레이아웃입니다.

```
┌────────────┬─────────────────────────────┐
│ [설정 폼]  │  알파 팩터 (N개)             │
│            │  ┌─────────────────────────┐ │
│ [진행률]   │  │ 팩터 테이블             │ │
│            │  │ (이름, 수식, IC, ...)    │ │
│ [이력]     │  └─────────────────────────┘ │
│            │  ┌─────────────────────────┐ │
│            │  │ 팩터 상세 (선택 시)      │ │
│            │  └─────────────────────────┘ │
└────────────┴─────────────────────────────┘
```

### 3.1 마이닝 설정 (좌측 상단)

마이닝을 시작하려면 다음 항목을 설정합니다:

| 항목 | 설명 | 기본값 | 권장값 (초보) |
|------|------|--------|-------------|
| **실행명** | 이번 마이닝에 붙일 이름 | "Alpha Mining" | "첫번째 테스트" |
| **시장 맥락** | AI에게 알려줄 시장 상황 설명 | (비어있음) | "한국 주식시장 대형주 모멘텀" |
| **유니버스** | 분석할 종목 그룹 (드롭다운) | KOSPI 200 | KOSPI 200 (권장) |
| **시작일** | 데이터 분석 시작 날짜 | 2024-01-01 | 2024-01-01 |
| **종료일** | 데이터 분석 끝 날짜 | 2025-12-31 | 2025-12-31 |
| **반복 횟수** | AI가 몇 개의 가설을 시도할지 | 5 | 3 (첫 시도) |
| **IC 임계값** | 팩터가 "발견"으로 인정되는 최소 IC | 0.03 | 0.03 |
| **직교성 필터** | 기존 팩터와의 최대 허용 상관계수 (슬라이더) | 0.70 | 0.70 |
| **PySR 사용** | Julia 기반 기호 회귀 (고급) | OFF | OFF |

> **유니버스 선택지:**
> - **KOSPI 200** (200종목) — 대형주 중심, **가장 권장** (횡단면 IC 통계적 유의성 확보)
> - **KOSDAQ 150** (150종목) — 코스닥 중대형주
> - **KRX 300** (300종목) — KOSPI+KOSDAQ 혼합
> - **전종목** — KOSPI+KOSDAQ 전체 (~3,800종목, 계산 시간 증가)

### 3.2 시장 맥락 입력 팁

시장 맥락은 AI가 팩터를 생성할 때 참고하는 힌트입니다. 구체적일수록 좋습니다.

**좋은 예시:**
- "한국 주식시장 대형주, RSI 과매도 반등 패턴 중심으로 탐색"
- "코스닥 반도체 관련주, 거래량 급증 시 모멘텀 포착"
- "변동성이 큰 장에서 역추세 전략, 볼린저밴드 활용"
- "KOSPI 200 전체를 대상으로 횡단면 모멘텀 + 밸류 팩터 탐색"

**나쁜 예시:**
- "돈 벌어줘" (너무 추상적)
- (비어있음) (기본 맥락 사용 — 가능하지만 비효율적)
- "삼성전자/SK하이닉스/NAVER 3종목" (종목 수가 너무 적어 횡단면 IC에 통계적 유의성 없음)

### 3.3 마이닝 시작

1. 설정을 완료한 뒤, **"알파 탐색 시작"** 버튼을 클릭합니다
2. 버튼이 **"마이닝 중..."**으로 바뀌며, 좌측에 **진행률 바**가 나타납니다
3. 각 반복마다 AI가 가설을 생성하고, 수식을 만들고, IC를 평가합니다
4. 반복 횟수 5회 기준, 약 **1~3분** 소요됩니다

### 3.4 진행 상태 읽기

마이닝이 진행되는 동안 좌측 패널에 진행률이 표시됩니다:

| 항목 | 설명 |
|------|------|
| **상태 배지** | PENDING(대기) → RUNNING(실행 중) → COMPLETED(완료) / FAILED(실패) |
| **진행률 바** | 파란색 바가 0%에서 100%까지 채워집니다 |
| **발견/평가** | "3개 발견 / 5개 평가" — IC 기준을 통과한 팩터 수 / 전체 시도 수 |

### 3.5 마이닝 이력 (좌측 하단)

과거에 실행한 마이닝 기록이 리스트로 표시됩니다:

- **초록 점** = 완료 (COMPLETED)
- **빨간 점** = 실패 (FAILED)
- **노란 점 (깜빡임)** = 실행 중 (RUNNING)

이력 항목을 **클릭하면** 해당 실행의 결과로 화면이 전환됩니다.
우측의 **"삭제"** 링크로 이력과 관련 팩터를 모두 삭제할 수 있습니다.

## 4. 팩터 테이블 읽기

마이닝이 완료되면 우측 패널에 **발견된 팩터 목록**이 테이블로 표시됩니다.

### 테이블 컬럼 설명

| 컬럼 | 의미 | 좋은 값 |
|------|------|---------|
| **이름** | 팩터의 자동 생성된 이름 (예: `alpha_0_0`) | — |
| **수식** | 팩터의 수학적 표현식 | 짧고 해석 가능한 수식 |
| **IC** | 정보계수 (팩터 ↔ 미래수익률 상관) | **0.03 이상** (높을수록 좋음) |
| **ICIR** | IC의 안정성 (IC / IC 표준편차) | **0.5 이상** |
| **Sharpe** | 샤프 비율 (위험 대비 수익) | **1.0 이상** |
| **상태** | 현재 팩터의 상태 | discovered → validated |
| **인과** | 인과 검증 결과 (Phase 2) | "검증됨" (초록) |
| **세대** | 진화 세대 (0=원본, 1+=변이) | — |

### 상태 배지 의미

| 상태 | 색상 | 의미 |
|------|------|------|
| **discovered** | 파란색 | 통계적으로 발견됨 (아직 검증 안 됨) |
| **validated** | 초록색 | 인과 검증까지 통과 (신뢰도 높음) |
| **mirage** | 빨간색 | 허위 상관으로 판명 (사용하면 안 됨) |

### 팩터 선택

테이블 행을 **클릭**하면 하단에 **팩터 상세 정보**가 펼쳐집니다.

## 5. 팩터 상세 보기

팩터를 선택하면 상세 패널이 나타납니다.

### 5.1 기본 정보

- **수식 코드 박스**: 팩터의 정확한 수학식이 코드 형태로 표시됩니다
  - 예: `rsi * volume_ratio + sma_20 / close`
- **가설**: AI가 이 팩터를 생성할 때의 추론 과정
  - 예: "RSI가 과매도 구간일 때 거래량이 증가하면 반등 가능성이 높다"

### 5.2 메트릭 그리드 (6개 지표)

```
┌───────────┐ ┌───────────┐ ┌───────────┐
│ IC Mean   │ │ IC Std    │ │ ICIR      │
│ 0.0523    │ │ 0.0045    │ │ 11.62     │
└───────────┘ └───────────┘ └───────────┘
┌───────────┐ ┌───────────┐ ┌───────────┐
│ Sharpe    │ │ MDD       │ │ Turnover  │
│ 8.34      │ │ -0.0823   │ │ 0.23      │
└───────────┘ └───────────┘ └───────────┘
```

**각 지표 해석:**

| 지표 | 의미 | 판단 기준 |
|------|------|----------|
| **IC Mean** | 팩터 값과 다음날 수익률의 평균 상관계수 | 0.03+ 양호, 0.05+ 우수, 0.10+ 탁월 |
| **IC Std** | IC의 변동성 (낮을수록 안정적) | 0.10 이하 양호 |
| **ICIR** | IC Mean / IC Std (IC의 일관성) | 0.5+ 양호, 1.0+ 우수 |
| **Sharpe** | 위험 조정 수익률 | 1.0+ 양호, 2.0+ 우수 |
| **MDD** | 최대 낙폭 (마이너스가 클수록 위험) | -0.10 이내 양호 |
| **Turnover** | 포트폴리오 회전율 (거래 빈도) | 0.3 이하 저비용 |

### 5.3 하단 액션 버튼

| 버튼 | 기능 | 언제 사용? |
|------|------|----------|
| **인과 검증** | DoWhy 4단계 인과 분석 실행 | 유망한 팩터의 진위 확인 시 |
| **이 팩터로 백테스트** | 팩터 기반 매매 전략 시뮬레이션 | 실제 수익률 확인 시 |

## 6. 인과 검증 (Phase 2)

### 왜 인과 검증이 필요한가?

IC가 높은 팩터라도 **허위 상관(Factor Mirage)**일 수 있습니다.

예를 들어:
- "아이스크림 판매량"과 "익사 사고 건수"는 높은 상관관계를 보이지만, 실제 원인은 "여름 기온"
- 마찬가지로 알파 팩터도 시장 전체 움직임(교란 변수)에 의한 착시일 수 있음

인과 검증은 이런 착시를 걸러냅니다.

### 6.1 인과 검증 실행 방법

1. 팩터 상세 화면에서 **"인과 검증"** 버튼을 클릭합니다
2. 안내 문구가 표시됩니다: *"인과 검증 진행 중... (DoWhy 4단계 분석, 1분 이상 소요될 수 있습니다)"*
3. 검증이 완료되면 결과가 표시됩니다

### 6.2 검증 결과 읽기

검증 완료 후 4가지 결과가 표시됩니다:

| 항목 | 의미 | 통과 기준 |
|------|------|----------|
| **인과 효과** | 팩터가 수익률에 미치는 실제 영향 크기 | 0이 아닌 유의미한 값 |
| **p-value** | 통계적 유의성 (낮을수록 신뢰) | **0.05 미만** |
| **Placebo 테스트** | 가짜 데이터로 바꿔도 효과가 사라지는지 | **"통과"** (초록색) |
| **Random Cause 테스트** | 랜덤 교란 변수를 추가해도 결과가 유지되는지 | **"통과"** (초록색) |
| **국면 생존력** | 시장 국면(전반/후반)이 바뀌어도 효과가 유지되는지 | **"통과"** (초록색) |

### 6.3 인과 DAG 그래프

결과 아래에 **인과 관계 그래프(DAG)**가 표시됩니다:

```
  시장수익률  시장변동성  기준금리
      \          |          /
       \         |         /
        ↘        ↓        ↙
          [ 알파 팩터 ] ──→ [ 미래 수익률 ]
              ↑
         [ 섹터 ]
```

- **초록색 화살표**: 인과 관계 유효 (통과)
- **빨간색 화살표**: 인과 관계 무효 (미라지)
- **회색 화살표**: 교란 변수의 영향 경로

### 6.4 국면 생존력 (Regime Shift) 검증

인과 검증의 5번째 단계로, **시장 국면 변화에도 팩터 효과가 유지되는지** 검증합니다.

**원리:**
1. 분석 기간 데이터를 시간 기준으로 **전반/후반**으로 분할
2. 각 반기에서 독립적으로 **ATE(Average Treatment Effect)** 추정
3. 전반과 후반의 ATE **부호(+/-)가 동일**하면 통과

**왜 중요한가:**
- 2024년 상반기에 유효했던 팩터가 하반기에 **부호가 뒤집히면**, 그 팩터는 특정 시장 국면에만 작동하는 것
- 국면 변화에 취약한 팩터로 실거래하면 **급격한 손실**을 입을 수 있음
- 전반/후반 모두 같은 방향의 효과를 보이는 팩터가 **장기적으로 신뢰할 수 있음**

**결과 표시:**
- 전반 ATE와 후반 ATE 수치가 각각 표시됨
- 부호가 같으면 "통과" (초록), 다르면 "실패" (빨간)

### 6.5 검증 후 배지 변화

검증 후 팩터 테이블의 "인과" 컬럼이 업데이트됩니다:

| 배지 | 색상 | 의미 |
|------|------|------|
| **미검증** | 회색 | 아직 검증하지 않음 |
| **검증됨** | 초록색 | 인과적으로 유효 — 실전 사용 가능 |
| **미라지** | 빨간색 | 허위 상관 — 사용하면 안 됨 |

## 7. 팩터로 백테스트 실행

유망한 팩터를 발견하면 실제 매매 시뮬레이션을 해볼 수 있습니다.

### 7.1 백테스트 시작

1. 팩터 상세 화면에서 **"이 팩터로 백테스트"** 버튼을 클릭합니다
2. 알림이 뜹니다: *"백테스트가 시작되었습니다. Run ID: xxx. 백테스트 페이지에서 결과를 확인하세요."*
3. **백테스트 페이지** (/backtest)로 이동하여 결과를 확인합니다

### 7.2 기본 백테스트 설정

자동으로 설정되는 기본값:

| 항목 | 기본값 | 의미 |
|------|--------|------|
| **매수 임계값** | 0.0 | 팩터 값 > 0 이면 매수 |
| **매도 임계값** | 0.0 | 팩터 값 < 0 이면 매도 |
| **시작일** | 2024-01-01 | — |
| **종료일** | 2025-12-31 | — |
| **초기 자본** | 1억 원 | — |
| **포지션 비중** | 10% | 한 종목당 자본의 10% |
| **최대 포지션** | 10개 | 동시 보유 최대 10종목 |

### 7.3 백테스트 결과 확인

백테스트 페이지에서 확인할 수 있는 항목:

- **수익 곡선**: 자산 변동 그래프
- **성과 카드**: 총 수익률, MDD, 승률, 프로핏 팩터
- **거래 내역**: 개별 매수/매도 기록

## 8. 공장 탭 — 자율 알파 팩토리 (Phase 3)

### 개념

"공장"은 사용자가 자고 있는 동안에도 AI가 **자동으로** 팩터를 탐색하는 기능입니다.

- 설정한 시간 간격(예: 6시간)마다 자동으로 마이닝 사이클 실행
- 발견된 팩터끼리 **유전 교차(Crossover)**를 통해 새로운 팩터 생성
- 과거 성공/실패 경험을 **벡터 메모리(RAG)**로 학습하여 점점 더 똑똑해짐

### 8.1 화면 구성

공장 탭은 위에서 아래로 세로 배치입니다:

```
┌─────────────────────────────────────────┐
│ 알파 팩토리 [가동 중/중지] [시작/중지]  │
│ 사이클: 5회 완료 | 발견: 23개           │
│ 진행률: ████████░░ 80%                  │
│ 마지막: 2026-03-02 14:30:00            │
└─────────────────────────────────────────┘

(중지 상태일 때만) 팩토리 설정 폼

┌─────────────────────────────────────────┐
│ 팩터 계보 (세대별 트리)                  │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 발견된 팩터 (N개) — 팩터 테이블          │
└─────────────────────────────────────────┘
```

### 8.2 팩토리 시작하기

1. **"공장"** 탭을 클릭합니다
2. 설정 폼에서 파라미터를 조정합니다 (아래 표 참조)
3. **"시작"** 버튼을 클릭합니다

### 8.3 팩토리 설정 항목

| 항목 | 설명 | 기본값 | 초보 권장 |
|------|------|--------|----------|
| **시장 맥락** | AI에게 전달할 탐색 방향 | (비어있음) | "한국 주식시장 알파 팩터 탐색" |
| **유니버스** | 분석 대상 종목 그룹 (드롭다운) | KOSPI 200 | KOSPI 200 (권장) |
| **시작일** | 데이터 시작일 | 2024-01-01 | 2024-01-01 |
| **종료일** | 데이터 종료일 | 2025-12-31 | 2025-12-31 |
| **사이클 간격** | 몇 분마다 반복할지 | 360분 (6시간) | 360 |
| **반복/사이클** | 사이클당 AI 시도 횟수 | 5 | 3 |
| **IC 임계값** | 발견 기준 IC | 0.03 | 0.03 |
| **직교성 필터** | 기존 팩터와의 최대 허용 상관계수 | 0.70 | 0.70 |
| **유전 교차** | ON/OFF 토글 | ON | ON |
| **인과 검증** | ON/OFF 토글 | OFF | OFF (시간 절약) |

### 8.4 팩토리 상태 읽기

팩토리가 가동 중이면 상단 카드에 실시간 정보가 표시됩니다:

| 항목 | 의미 |
|------|------|
| **배지: "가동 중"** | 파란색 배지 — 팩토리가 작동 중 |
| **사이클: N회 완료** | 지금까지 완료된 마이닝 사이클 수 |
| **발견 팩터: N개** | 모든 사이클에서 발견된 총 팩터 수 |
| **현재 진행률: N%** | 현재 사이클의 진행 상태 |
| **마지막 사이클** | 가장 최근 사이클 완료 시각 |

진행률 아래에 **파란 진행 바**가 표시되며, 5초마다 자동 갱신됩니다.

### 8.5 팩토리 중지

- **"중지"** 버튼을 클릭하면 팩토리가 중지됩니다
- 현재 진행 중인 사이클이 있으면 완료 후 중지됩니다
- 중지 후 설정을 변경하고 다시 시작할 수 있습니다

### 8.6 유전 교차 (Crossover) 이해하기

유전 교차가 **ON**이면:

1. 각 사이클에서 발견된 팩터들을 "부모"로 선택
2. 두 부모의 수식에서 일부를 교환하여 "자식" 팩터 생성
3. 자식 팩터도 IC 평가를 거쳐, 기준 통과 시 발견 목록에 추가

예시:
- 부모 A: `rsi * volume_ratio`
- 부모 B: `sma_20 / close + macd_hist`
- 자식: `rsi * (sma_20 / close)` (A의 rsi와 B의 sma_20/close 조합)

이를 통해 단순 AI 생성보다 다양하고 예상치 못한 팩터가 탄생합니다.

## 9. 팩터 계보 트리

### 개념

공장에서 유전 교차가 활성화되면, 팩터들 사이에 부모-자식 관계가 생깁니다. 계보 트리는 이 관계를 시각적으로 보여줍니다.

### 9.1 트리 읽는 법

```
      [세대 0: 원본 팩터]
    ┌─────────┐  ┌─────────┐
    │ alpha_0 │  │ alpha_1 │
    │ IC=0.04 │  │ IC=0.05 │
    └────┬────┘  └────┬────┘
         │            │
         └──────┬─────┘
                ↓
      [세대 1: 교차 자식]
    ┌──────────────┐
    │ crossover_0  │
    │ IC=0.06      │
    └──────────────┘
```

- **Y축** = 세대 (위에서 아래로 진화)
- **노드** = 개별 팩터 (이름 + IC + 세대 번호)
- **화살표** = 부모 → 자식 관계

### 9.2 노드 색상 의미

| 색상 | 조건 | 의미 |
|------|------|------|
| **초록색** | IC >= 0.05 | 우수한 팩터 |
| **파란색** | IC >= 0.03 | 양호한 팩터 |
| **주황색** | IC < 0.03 | 미흡한 팩터 |
| **회색** | IC 없음 | 아직 평가 안 됨 |

### 9.3 노드 클릭

트리의 노드를 클릭하면 해당 팩터의 **상세 정보**가 하단에 표시됩니다.

## 10. 포트폴리오 탭 — 복합 팩터 (Phase 3)

### 개념

개별 팩터 하나만으로는 한계가 있습니다. 여러 팩터를 조합하면:

- **분산 효과**: 한 팩터가 틀려도 다른 팩터가 보완
- **안정성 향상**: IC의 변동성 감소
- **시너지**: 서로 다른 관점의 팩터가 합쳐져 더 강력한 신호

### 10.1 화면 구성

```
┌─────────────────────────────────────────┐
│ 팩터 선택 (3개 선택됨)                   │
│ ☑ alpha_0          [IC 0.0523]          │
│ ☐ alpha_1          [IC 0.0412]          │
│ ☑ alpha_2          [IC 0.0378]          │
│ ☑ crossover_0      [IC 0.0612]          │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 상관행렬 히트맵                          │
│ (선택된 팩터 간 상관관계)                 │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 복합 팩터 이름: [Composite Alpha     ]  │
│ 가중 방법: ● IC 가중  ○ 동일 가중       │
│ [         복합 팩터 생성 (3개)        ]  │
└─────────────────────────────────────────┘
```

### 10.2 사용 방법 (단계별)

#### Step 1: 팩터 선택

- 체크박스로 합성할 팩터를 **2개 이상** 선택합니다
- 각 팩터 옆에 **IC 배지**가 표시되어 성능을 바로 비교할 수 있습니다
- "single" 타입 팩터만 표시됩니다 (이미 합성된 복합 팩터는 제외)

#### Step 2: 상관행렬 확인

2개 이상 선택하면 자동으로 **상관행렬 히트맵**이 나타납니다.

**핵심 원칙: 상관관계가 낮은 팩터끼리 합성하세요.**

- 상관 0.8 이상 = 거의 같은 팩터 → 합성 효과 없음
- 상관 0.3~0.5 = 적당한 독립성 → 좋은 후보
- 상관 0.0~0.2 = 거의 무관 → 최적의 분산 효과

#### Step 3: 가중 방법 선택

| 방법 | 설명 | 언제 사용? |
|------|------|----------|
| **IC 가중** | IC가 높은 팩터에 더 높은 가중치 | 팩터 간 성능 차이가 클 때 (권장) |
| **동일 가중** | 모든 팩터에 동일한 가중치 (1/N) | 팩터 간 성능이 비슷할 때 |

#### Step 4: 이름 입력 + 생성

1. 복합 팩터의 이름을 입력합니다 (기본: "Composite Alpha")
2. **"복합 팩터 생성 (N개)"** 버튼을 클릭합니다
3. 생성 완료 시 알림이 뜹니다: *"복합 팩터 생성 완료! IC: 0.0678"*

생성된 복합 팩터는 팩터 테이블에 추가되며, 일반 팩터와 동일하게 백테스트할 수 있습니다.

## 11. 상관행렬 히트맵

### 읽는 법

```
           alpha_0  alpha_1  alpha_2
alpha_0  [ 1.00    0.23     0.65  ]
alpha_1  [ 0.23    1.00     0.11  ]
alpha_2  [ 0.65    0.11     1.00  ]
```

- **대각선**: 항상 1.00 (자기 자신과의 상관)
- **빨간색**: 음의 상관 (하나가 오르면 다른 하나는 내림)
- **흰색**: 무상관 (서로 무관)
- **초록색**: 양의 상관 (같이 움직임)

### 복합 팩터 선택 가이드

| 상관 값 | 색상 | 해석 | 합성 적합도 |
|---------|------|------|-----------|
| -0.5 ~ -1.0 | 빨간색 | 반대로 움직임 | 매우 적합 (헤지 효과) |
| -0.2 ~ 0.2 | 흰색 | 거의 무관 | 적합 (분산 효과) |
| 0.3 ~ 0.5 | 연한 초록 | 약한 양의 상관 | 보통 |
| 0.7 ~ 1.0 | 진한 초록 | 거의 같은 움직임 | 부적합 (중복) |

## 12. 응용: 실전 워크플로우

### 워크플로우 A: 빠른 팩터 탐색 (30분)

**목표**: 빠르게 유망한 팩터 1~2개 찾기

1. **탐색 탭** → 시장 맥락 입력 → 반복 3회 → "알파 탐색 시작"
2. 완료 후 IC 높은 순으로 팩터 확인
3. IC 0.03 이상 팩터에 **"인과 검증"** 실행
4. "검증됨" 배지를 받은 팩터로 **"이 팩터로 백테스트"**
5. 백테스트 결과 확인 → 수익률/MDD 만족 시 실전 고려

### 워크플로우 B: 자율 팩토리 운영 (장기)

**목표**: 며칠에 걸쳐 다양한 팩터를 자동 탐색

1. **공장 탭** → 설정 입력:
   - 시장 맥락: "한국 대형주 + 코스닥 성장주 혼합"
   - 유니버스: KOSPI 200 (권장) 또는 KRX 300
   - 사이클 간격: 360분 (6시간)
   - 유전 교차: ON
2. **"시작"** 클릭 → 팩토리 방치
3. 다음 날 확인: 팩터 목록에 새 팩터 추가됨
4. 계보 트리에서 세대별 진화 확인
5. IC 높은 팩터 선별 → 인과 검증 → 백테스트

### 워크플로우 C: 복합 팩터 구성 (고급)

**목표**: 검증된 팩터들을 합성하여 안정적인 투자 신호 만들기

1. 워크플로우 A/B로 "검증됨" 팩터 5개 이상 확보
2. **포트폴리오 탭** → 팩터 5개 체크
3. 상관행렬 히트맵에서 상관 0.5 이상인 쌍 제외
4. 3~4개 선택 → IC 가중 → "복합 팩터 생성"
5. 복합 팩터의 IC를 개별 팩터와 비교 (더 높아야 의미 있음)
6. 복합 팩터로 백테스트 → 개별 팩터 백테스트와 비교

## 13. 고급: 파라미터 튜닝 전략

### 13.1 IC 임계값 조정

| IC 임계값 | 효과 | 권장 상황 |
|-----------|------|----------|
| **0.01** | 매우 관대 — 많은 팩터 발견, 품질 낮음 | 탐색 초기, 다양성 확보 |
| **0.03** (기본) | 균형 — 적당한 수와 품질 | 일반적 사용 |
| **0.05** | 엄격 — 소수 고품질 팩터만 | 실전 투입 전 최종 선별 |
| **0.10** | 매우 엄격 — 거의 발견 안 됨 | 연구 목적 |

### 13.2 반복 횟수 vs 품질

| 반복 횟수 | 소요 시간 | 특성 |
|----------|----------|------|
| 3 | ~1분 | 빠른 탐색, 팩터 0~2개 |
| 5 (기본) | ~2분 | 균형, 팩터 1~3개 |
| 10 | ~5분 | 심화 탐색, 팩터 2~5개 |
| 20 | ~10분 | 최대 탐색, 팩터 3~8개 |

### 13.3 시장 맥락 고급 활용

**맥락별 팩터 성격 유도:**

| 맥락 예시 | 생성되는 팩터 성격 |
|----------|------------------|
| "RSI 과매도 반등" | 역추세, 평균회귀 팩터 |
| "거래량 급증 모멘텀" | 추세추종, 브레이크아웃 팩터 |
| "볼린저밴드 수축 후 확장" | 변동성 팩터 |
| "이동평균 크로스 장기 추세" | 트렌드 팩터 |
| "MACD 히스토그램 다이버전스" | 모멘텀 전환 팩터 |

**여러 맥락을 합치면 다양한 팩터 생성:**
```
"한국 대형주, RSI 과매도 반등 + 거래량 브레이크아웃 + 이동평균 크로스를
복합적으로 고려하는 팩터"
```

### 13.4 유니버스 선택 전략

> **핵심 원칙:** 횡단면 IC(Cross-sectional IC)의 통계적 유의성을 확보하려면
> **최소 50개 이상** 종목이 포함된 유니버스를 사용해야 합니다.
> 3~10개 종목으로 계산한 IC는 표본이 너무 작아 의미가 없습니다.

| 유니버스 | 종목 수 | 특성 | 권장 용도 |
|----------|---------|------|----------|
| **KOSPI 200** | ~200개 | 대형주 중심, 유동성 풍부 | **기본 권장** — 안정적 IC 추정 |
| **KOSDAQ 150** | ~150개 | 코스닥 중대형 성장주 | 성장주/바이오 특화 팩터 |
| **KRX 300** | ~300개 | KOSPI+KOSDAQ 혼합 | 범용 팩터, 최대 분산 |
| **전종목** | ~3,800개 | 전체 시장 | 연구 목적 (계산 시간 주의) |

**유니버스별 기대 효과:**
- **KOSPI 200**: IC 변동이 적고 안정적. 실전 투자 가능한 유동성 확보. 가장 추천.
- **KOSDAQ 150**: 변동성이 크므로 IC 절댓값이 높게 나올 수 있지만 불안정.
- **KRX 300**: KOSPI+KOSDAQ 혼합으로 다양한 시장 상황을 반영. 범용 팩터 탐색에 적합.
- **전종목**: 소형주/비유동주 포함으로 실전 적용이 어려울 수 있음. 이론적 연구용.

### 13.5 유전 교차 최적 조건

유전 교차가 효과적인 경우:
- 발견된 팩터가 **3개 이상**일 때 (교차 대상 풀이 충분해야)
- 팩터들의 **수식 구조가 다양**할 때 (RSI 기반 + 거래량 기반 등)
- **IC가 비슷한 수준**의 팩터가 여럿일 때

유전 교차가 비효율적인 경우:
- 팩터가 1~2개뿐일 때
- 모든 팩터가 거의 같은 수식일 때

### 13.6 복합 팩터 고급 전략

**전략 1: 팩터 Zoo 정리**

10개 이상 팩터가 있으면:
1. 상관행렬로 클러스터링 (상관 0.7 이상 = 같은 그룹)
2. 각 그룹에서 IC 최고인 대표 팩터 1개씩 선택
3. 대표 팩터들로 복합 팩터 생성

**전략 2: IC 가중 vs 동일 가중 비교**

같은 팩터 셋으로 두 가지 방법을 모두 시도하여 IC를 비교:
- IC 가중이 높으면 → 소수 팩터에 집중이 유리
- 동일 가중이 높으면 → 분산이 유리

### 13.7 PySR 과적합 방지

PySR(기호 회귀)을 사용하면 AI가 자유롭게 수식 구조를 탐색하므로, **과도하게 복잡한 수식(과적합)**이 발생할 수 있습니다. 이를 방지하기 위한 2가지 파라미터가 있습니다:

| 파라미터 | 설명 | 기본값 | 튜닝 가이드 |
|----------|------|--------|------------|
| **수식 최대 길이 (Max Size)** | 수식에 포함될 수 있는 최대 노드 수 | 15 | 작을수록 간결한 수식 (5~10: 보수적, 20~50: 탐색적) |
| **간결성 압박 (Parsimony)** | 수식 길이에 대한 페널티 계수 | 0.03 | 클수록 짧은 수식 선호 (0.1+: 매우 간결, 0.001: 제약 없음) |

**권장 설정:**
- **실전 투입용**: Max Size 10, Parsimony 0.1 — 해석 가능한 간결한 수식
- **탐색/연구용**: Max Size 25, Parsimony 0.01 — 복잡한 비선형 관계 발견
- **기본값** (Max Size 15, Parsimony 0.03): 균형 잡힌 설정

> PySR 고급 설정은 "PySR 기호 회귀 사용" 토글을 ON으로 설정해야만 표시됩니다.

**전략 3: Walk-Forward 검증**

1. 2024-01-01 ~ 2024-12-31 데이터로 팩터 탐색 (in-sample)
2. 발견된 팩터로 2025-01-01 ~ 2025-06-30 백테스트 (out-of-sample)
3. out-of-sample에서도 IC가 유지되는 팩터만 실전 투입

## 14. 트러블슈팅

### 마이닝이 실패(FAILED)합니다

**원인 1**: 데이터 부족
- 확인: 시작일~종료일 기간에 해당 종목의 캔들 데이터가 DB에 있는지 확인
- 해결: 더 넓은 기간 설정 또는 종목 변경

**원인 2**: API 키 미설정
- 확인: 백엔드의 `ANTHROPIC_API_KEY` 환경변수 확인
- 해결: `.env` 파일에 유효한 API 키 설정

**원인 3**: 유니버스 데이터 없음
- 확인: 선택한 유니버스의 종목 캔들 데이터가 DB에 시딩되어 있는지 확인
- 해결: `python -m scripts.seed_candles` 실행하여 캔들 데이터 시딩

### 팩터가 하나도 발견되지 않습니다

**원인 1**: IC 임계값이 너무 높음
- 해결: IC 임계값을 0.02 또는 0.01로 낮춰보세요

**원인 2**: 반복 횟수 부족
- 해결: 반복 횟수를 10~20으로 높이세요

**원인 3**: 유니버스 종목 수가 부족하거나 기간이 짧음
- 해결: KOSPI 200 등 50개 이상 종목이 포함된 유니버스 선택, 기간 6개월 이상 권장

### 인과 검증이 오래 걸립니다

- DoWhy 인과 검증은 팩터당 **30초~2분** 소요됩니다
- 서버 CPU 성능에 따라 더 길어질 수 있습니다
- 팩토리에서 인과 검증을 ON하면 사이클이 매우 느려집니다 → OFF 권장

### 복합 팩터 생성 시 에러

**"유효한 팩터가 2개 미만입니다"**
- 선택한 팩터의 수식을 파싱할 수 없는 경우 발생
- 다른 팩터 조합을 시도하세요

**"데이터 행이 부족합니다"**
- 팩터 계산 후 유효한 데이터가 30행 미만
- 더 넓은 기간의 데이터가 필요합니다

### 상관행렬이 표시되지 않습니다

- 팩터를 **2개 이상** 선택해야 합니다
- 서버에 캔들 데이터가 충분히 있어야 합니다

## 15. 용어 사전

| 용어 | 영문 | 설명 |
|------|------|------|
| **알파** | Alpha | 시장 평균을 초과하는 초과 수익 |
| **팩터** | Factor | 수익률을 예측하는 데 사용되는 변수/수식 |
| **IC** | Information Coefficient | 횡단면 정보계수(Cross-sectional IC). 특정 시점에서 전체 종목의 팩터 값과 T+1 수익률의 순위 상관계수(Spearman Rank). 이를 전체 기간에 대해 평균한 것이 IC Mean. 통계적 유의성을 위해 최소 50개 이상 종목이 필요함 |
| **ICIR** | IC Information Ratio | IC의 평균 / IC의 표준편차. IC의 안정성 지표 |
| **MDD** | Maximum Drawdown | 고점 대비 최대 하락폭 (%) |
| **Sharpe** | Sharpe Ratio | 위험 조정 수익률 = 초과수익률 / 표준편차 |
| **Turnover** | Turnover | 포트폴리오 회전율 (높으면 거래 비용 증가) |
| **미라지** | Factor Mirage | 통계적 우연에 의한 허위 상관 (가짜 알파) |
| **인과 검증** | Causal Validation | DoWhy 프레임워크로 진짜 인과 관계인지 확인 |
| **교란 변수** | Confounder | 팩터와 수익률 모두에 영향을 주는 제3의 변수 |
| **Placebo 테스트** | Placebo Test | 팩터 값을 가짜로 바꿔도 효과가 사라지는지 확인 |
| **국면 생존력** | Regime Shift Survival | 데이터를 전반/후반으로 분할했을 때 인과 효과의 부호가 일관되는지 검증 |
| **직교성 필터** | Orthogonality Filter | 신규 팩터가 기존 팩터와 피어슨 상관계수 임계치 미만인지 검사하여 중복 팩터를 제거 |
| **유니버스** | Universe | 분석 대상 종목 그룹 (KOSPI 200, KOSDAQ 150, KRX 300, 전종목) |
| **유전 교차** | Crossover | 두 수식의 일부를 교환하여 새로운 수식 생성 |
| **변이** | Mutation | 수식의 일부를 랜덤으로 변경 |
| **세대** | Generation | 진화 과정에서의 세대 번호 (0=원본) |
| **RAG** | Retrieval-Augmented Generation | 과거 경험을 검색하여 AI 생성에 활용 |
| **벡터 메모리** | Vector Memory | 과거 팩터를 임베딩으로 저장하여 유사도 검색 |
| **복합 팩터** | Composite Factor | 여러 단일 팩터를 가중 합산한 팩터 |
| **IC 가중** | IC-Weighted | IC가 높은 팩터에 더 높은 가중치 부여 |
| **동일 가중** | Equal Weight | 모든 팩터에 동일한 가중치 (1/N) |
| **상관행렬** | Correlation Matrix | 팩터 간 상관계수를 행렬로 표시 |
| **DAG** | Directed Acyclic Graph | 인과 관계를 화살표로 표현한 그래프 |
| **p-value** | p-value | 귀무가설이 맞을 확률. 0.05 미만이면 통계적 유의 |
| **RSI** | Relative Strength Index | 상대강도지수 (0~100, 과매수/과매도 판단) |
| **SMA** | Simple Moving Average | 단순 이동평균 |
| **MACD** | Moving Average Convergence Divergence | 이동평균 수렴확산 |
| **볼린저밴드** | Bollinger Bands | 이동평균 ± 표준편차 밴드 |
| **ATR** | Average True Range | 평균 진폭 (변동성 지표) |
