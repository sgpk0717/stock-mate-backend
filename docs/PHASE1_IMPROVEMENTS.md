# Phase 1 알파 탐색 — 보강 필요 사항

Phase 1 구현 완료 후 사용자 관점에서 식별된 UX/기능 개선점.
Phase 2~4 진행 후 또는 병행하여 처리.

---

## 1. 팩터 → 백테스트 원클릭 연결 강화

**현재**: "이 팩터로 백테스트" 버튼이 고정값(buy_threshold=0, sell_threshold=0)으로 바로 실행
**개선**: 모달/패널에서 매수/매도 임계값, 기간, 종목, 포지션 사이즈를 설정할 수 있는 폼 제공. 백테스트 완료 후 결과 페이지로 자동 이동.

## 2. 팩터 해석 시각화

**현재**: 수식(`log(volume_ratio) * (30 - rsi) / atr_14`)과 숫자 메트릭만 표시. 사용자가 의미를 직관적으로 파악하기 어려움.
**개선**:
- IC 시계열 차트 (일별 IC 값의 추이 — AlphaICChart 컴포넌트는 계획에 있으나 미구현)
- 팩터 값 히트맵: 시간축 × 팩터값 컬러맵으로 매수/매도 타이밍 시각화
- Claude 자연어 해석: 수식을 Claude에게 넘겨 "이 팩터는 ~할 때 높아지며, ~를 의미합니다" 설명 생성

## 3. 기존 전략 조건에 팩터 추가

**현재**: 알파 팩터를 백테스트 엔진의 `alpha_*` 지표로 등록하는 브릿지는 있지만, 프론트에서 기존 전략 조건에 팩터를 추가하는 UI가 없음.
**개선**: BacktestPage의 조건 설정 UI에서 "알파 팩터" 드롭다운 → 발견된 팩터 목록에서 선택 → 기존 RSI, MACD 등과 AND/OR 조합 가능.

## 4. 원클릭 실전 적용 흐름

**현재**: 팩터 발견 → 백테스트까지만. 모의투자/실거래 전환 연결 없음.
**개선**: 백테스트 결과 페이지에서 "이 전략으로 모의투자 시작" → TradingContext 자동 생성 → /trading 페이지 이동. (기존 BacktestPage의 전환 버튼 패턴 재사용)

## 5. 마이닝 결과 요약 대시보드

**현재**: 팩터 목록 테이블만 있음. 전체 마이닝 세션의 성과를 한눈에 볼 수 없음.
**개선**: 마이닝 완료 후 요약 카드 — "N개 수식 평가, M개 팩터 발견, 최고 IC=0.08, 평균 세대=2.3" 등.

## 6. 팩터 비교 기능

**현재**: 팩터를 하나씩 클릭해서 상세 보기만 가능.
**개선**: 체크박스로 2~3개 팩터 선택 → 나란히 비교 (수식, IC, Sharpe, MDD). Phase 3의 상관행렬/복합 팩터 빌더의 사전 단계.
